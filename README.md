# YouTube MP3 Backup

A small Windows PowerShell app for backing up audio from YouTube videos, playlists, or channels you own or have permission to archive.

It uses:

- `yt-dlp` to download the best available audio stream
- `ffmpeg` to convert/extract high quality MP3 files
- `deno` as the local JavaScript runtime needed by current YouTube extraction
- `C:\_Media\MP3` as the default output folder

## Quick Start

1. Open PowerShell in this folder.
2. Run:

   ```powershell
   .\setup.ps1
   ```

3. Start the app:

   ```powershell
   .\Start-YouTubeMp3.ps1
   ```

Or run the pinned-friendly executable:

```powershell
.\YouTubeMp3Backup.exe
```

## CLI Usage

Download one video, playlist, or channel URL:

```powershell
.\Download-YouTubeMp3.ps1 -Url "https://www.youtube.com/watch?v=VIDEO_ID"
```

Download several URLs:

```powershell
.\Download-YouTubeMp3.ps1 -Url "https://www.youtube.com/watch?v=VIDEO_1","https://www.youtube.com/watch?v=VIDEO_2"
```

Or put one URL per line in a text file:

```powershell
.\Download-YouTubeMp3.ps1 -UrlFile ".\links.txt"
```

Choose a custom output folder:

```powershell
.\Download-YouTubeMp3.ps1 -Url "https://www.youtube.com/@YourChannel/videos" -OutputFolder "D:\AudioBackups"
```

Use browser cookies for YouTube sign-in checks:

```powershell
.\Download-YouTubeMp3.ps1 -UrlFile ".\links.txt" -CookieBrowser edge
```

Archive mode writes a download archive file so reruns skip items already downloaded:

```powershell
.\Download-YouTubeMp3.ps1 -Url "https://www.youtube.com/@YourChannel/videos" -Archive
```

## Options

- **Archive mode** creates `download-archive.txt` in the output folder. Future runs check that file and skip videos that were already downloaded.
- **Single video only** passes `--no-playlist` to `yt-dlp`. Keep it on for normal watch links, especially links with `list=` or `start_radio=1`. Turn it off when you intentionally want a playlist or channel backup.
- **Use browser cookies** passes `--cookies-from-browser` to `yt-dlp`. Choose the browser where you are already signed into YouTube. Cookies are read locally and should not be pasted into chat or committed to the repo.

## EXE and Signing

Build or rebuild the executable:

```powershell
.\Build-Exe.ps1
```

The executable uses `app-icon.ico`, a red play-button icon generated by:

```powershell
.\New-AppIcon.ps1
```

The build script signs `YouTubeMp3Backup.exe` with a local self-signed code-signing certificate named `Chase Cole Software`. Windows can verify the signature, but it will not be publicly trusted like a paid code-signing certificate.

To trust that local certificate for your Windows user:

```powershell
.\Trust-LocalCodeSigningCert.ps1
```

## Notes

- MP3 is lossy. YouTube audio is usually already compressed, so this app uses `-q:a 0` for high quality VBR MP3, but it cannot create quality that was not in the source.
- For a true backup of the original downloaded audio stream, use yt-dlp directly with `-f bestaudio` and skip MP3 conversion.
- Only download content you own, control, or have permission to archive.
